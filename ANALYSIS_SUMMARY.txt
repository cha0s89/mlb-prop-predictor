THE PROBLEM
================================================================================

v009 weights configuration claimed to achieve 58.4% accuracy on FS MORE by 
increasing the variance ratio from 4.0 to 5.0. However, the actual backtest 
data shows:

  FS MORE: 39.5% accuracy (NOT 58.4%)
  TB MORE: 36.0% accuracy
  FS LESS: 69.9% accuracy

The variance ratio change was PROPOSED but never actually applied and tested.

More critically: the OFFSETS in v009 are DESTROYING the MORE direction by 
over-correcting upward bias. Analysis reveals:

  FS offset 1.23 → Projects are pushed DOWN by 1.23 pts → MORE becomes pessimistic
  TB offset 0.78 → Projects are pushed DOWN by 0.78 pts → MORE becomes pessimistic
  Hits offset 0.48 → Projects are pushed DOWN by 0.48 pts → MORE becomes pessimistic

These offsets were supposed to correct for model optimism, but they're 
OVERCORRECTING by 2-3x.

================================================================================

Conducted comprehensive grid search over 600 offset configurations:

  Dimensions tested:
    - hitter_fantasy_score: [0.0, 0.3, 0.5, 0.8, 1.0, 1.23]
    - total_bases: [0.0, 0.2, 0.4, 0.6, 0.78]
    - hits: [0.0, 0.1, 0.2, 0.3, 0.48]
    - pitcher_strikeouts: [-0.58, -0.40, -0.20, 0.0]

  Key Finding: Reducing offsets improves MORE accuracy for all prop types

Best configuration found:
  FS offset 0.0 (from 1.23) → FS MORE 45.6% (from 39.5%, +6.1pp)
  TB offset 0.2 (from 0.78) → TB MORE 44.5% (from 36.0%, +8.5pp)
  Hits offset 0.0 (from 0.48) → Hits MORE 50.0% (from 27.7%, +22.3pp!)
  PK offset -0.58 (unchanged) → PK MORE 54.7% (from 56.1%, -1.4pp acceptable)

However, even the "best" configuration falls short of the 54% threshold for 
both MORE and LESS directions on multiple prop types. This indicates a 
DEEPER STRUCTURAL PROBLEM.

================================================================================
THE ROOT CAUSE
================================================================================

Analysis of actual vs projected values reveals systematic bias:

  Prop Type       Mean Actual   Mean Projected   Bias    Direction
  ──────────────────────────────────────────────────────────────────
  FS              6.50 pts      7.54 pts         +1.04   OVERESTIMATED
  Total Bases     1.37 bases    1.56 bases       +0.19   OVERESTIMATED
  Hits            0.83 hits     0.96 hits        +0.13   OVERESTIMATED
  PK Ks           4.84 Ks       5.50 Ks          +0.67   OVERESTIMATED

The model CONSISTENTLY OVERESTIMATES player performance across all prop types.

This is not a calibration problem fixable by offsets. Root causes likely include:

  1. Base rates in predictor.py may be from elite subsets (not population mean)
  2. Factor multipliers (park, matchup, platoon) may be too aggressive
  3. Statcast blending (xBA, xSLG) may be optimistic vs DraftKings line universe
  4. PrizePicks selection bias: lines are curated, not random samples
