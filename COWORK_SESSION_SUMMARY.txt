KEY FINDINGS
================================================================================

1. NON-PLAY PROBLEM (21-44% of predictions)
   - Batters who didn't actually bat are still in backtest (actual=0)
   - These create automatic LESS wins and MORE losses
   - Example: Max Muncy had 200+ season PA but 0 actual PA on April 2 game 1
   - Impact: MORE accuracy depressed by 10-25pp, LESS inflated by 10-25pp

2. UNDERPROJECTION NOT OVERPROJECTION
   - When analyzing played-games-only (actual > 0):
     * Fantasy Score: projection 7.54 vs actual 8.77 (underproject by 1.23)
     * Total Bases: projection 1.57 vs actual 2.35 (underproject by 0.77)
     * Hits: projection 0.96 vs actual 1.44 (underproject by 0.48)
     * Home Runs: projection 0.14 vs actual 1.07 (underproject by 0.93)
   - Only pitcher K overprojected (+0.57)
   - This is OPPOSITE of what v005 weights assumed!

3. DECIMAL BUG FIX ENABLED BETTER ANALYSIS
   - Commit 821f436 added _ensure_pct() for K%/BB% handling
   - Previous backtest had K%/BB% decimal bug
   - Forced fresh backtest run with corrected code
